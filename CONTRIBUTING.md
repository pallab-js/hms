# Contributing to HMS

We love your input! We want to make contributing to HMS as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## Development Process

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`npm test` and `cd src-tauri && cargo test`).
4. Ensure your code lints.
5. Issue that pull request!

## Pull Request Guidelines

- Include a clear description of the change.
- Reference any related issues.
- Provide screenshots or GIFs for UI changes if possible.
